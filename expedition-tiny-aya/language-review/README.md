# Language Review & Sign-off

Quality review of keyword mappings and transpiled code samples for each target language.

## Review Process

For each language (zh, es, ur):

1. **Keyword review** — Verify Legesher keyword mappings are correct and natural
2. **Sample review** — Review transpiled files for correctness and readability
3. **Model output review** — Spot-check model outputs in target language
4. **Sign-off** — Formal approval that transpiled code is ready for training

## Directory Structure

Each language has its own folder with review documents:

```
zh/  — Chinese review notes and sign-off
ur/  — Urdu review notes and sign-off
```
