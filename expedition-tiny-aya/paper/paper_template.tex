\begin{abstract}
Large language models (LLMs) have shown remarkable capabilities in multilingual tasks, but the impact of training on code written in native languages remains underexplored. This paper investigates whether fine-tuning multilingual models on native-language code improves their reasoning, world knowledge, and instruction following compared to English code or no code training. We conduct a systematic experiment using Tiny Aya, fine-tuning it on various conditions of code data in Chinese, Spanish, and Urdu. Evaluations on MGSM, X-CSQA, and XNLI benchmarks reveal that native-language code provides unique benefits, with keyword-swapped code showing initial gains and mixed native sources yielding further improvements. Our findings suggest that cultural and linguistic signals in native code enhance model performance beyond mere language exposure.
\end{abstract}

\section{Introduction}
Multilingual large language models (LLMs) are increasingly vital for global applications, yet their performance often lags in non-English languages due to imbalanced training data. Recent research has demonstrated that training on code can enhance reasoning and instruction-following abilities in LLMs \cite{code_or_not}. However, the question remains: does the natural language in which the code is written matter?

This paper explores the hypothesis that code composed in native languages carries unique cultural and linguistic signals that improves native language performance beyond what is achievable using English code. We investigate this through controlled experiments on TinyAya, a multilingual LLM, using an ``experimental ladder'' of conditions that progressively introduce native-language elements.

Our core research question is: Does multilingual code improve Tiny Aya's reasoning, world knowledge, and instruction following? We target Chinese (zh), Spanish (es), and Urdu (ur), evaluating on MGSM for math reasoning, X-CSQA for commonsense reasoning, and XNLI for natural language inference.

The experiment isolates variables: from no fine-tuning (baseline) to English code, keyword-swapped code, mixed native sources, and strictly native code. This ladder design allows us to measure incremental benefits of native-language exposure.

Our results show that native-language code yields significant improvements, with implications for developing more equitable multilingual AI systems.

\section{Literature Review}
The role of code in training LLMs has gained attention following \citep{code_or_not} study of ``To Code or Not to Code'', which found that pretraining on English Python code improves mathematical reasoning and instruction following. This work established code as a valuable training signal, but focused exclusively on English.

Multilingual programming languages represent an emerging area. Research on frameworks for localizing programming languages highlights the challenges and opportunities in adapting code to native linguistic structures \cite{localization_framework}. Additionally, studies on multilingual programming languages explore how non-English keywords and syntax can make coding more accessible \cite{multilingual_pl}.

However, a key gap remains: while code training benefits are established, the comparative impact of native-language code versus English code on multilingual model performance is underexplored. Our work addresses this by systematically comparing conditions that isolate linguistic and cultural elements in code training.

\section{Methodology}
We employ an experimental ladder design to isolate the impact of native-language code on multilingual LLM performance. The base model is \texttt{CohereLabs/tiny-aya-base} (3.35B parameters), fine-tuned using QLoRA 4-bit quantization on Kaggle T4 GPUs.

\subsection{Data Curation} Data was curated according to a quality hierarchy designed to capture progressively richer linguistic signals in code:

\begin{itemize} \item \textbf{Fully native code} (best): Code written entirely in the target language, including identifiers, syntax adaptations, and comments. \item \textbf{Function names, variable names, and comments in target language} (good): Code retains English syntax but incorporates localized naming and documentation. \item \textbf{Comments only in target language} (okay): Code remains structurally in English, but includes localized explanations. \end{itemize}

Datasets for each language were constructed from multiple sources:

\begin{itemize} \item Code extracted from \textbf{The Stack} dataset \cite{kocetkov2022stack3tbpermissively} for the target language. \item Native programming languages in the target languages (e.g., Wenyan and Qi for Chinese via GitHub and Gitee; Latino programming language for Spanish via GitHub). \item Transpiled Python programs into the target language using Legesher \cite{Edgar_Legesher_Because_language_2026}. \end{itemize}

Each language dataset consisted of a mixture of these sources, ensuring diversity in linguistic and structural representation. We also used static code analysis tools to verify the quality and correctness of the code samples, particularly for the fully native code condition.

The datasets were uploaded to Hugging Face \cite{legesher_language_decoded} for accessibility during fine-tuning.

\subsection{Experimental Conditions}
The conditions form a progressive ladder, each building on the previous to test incremental hypotheses:

\begin{enumerate}
\item \textbf{Baseline}: No fine-tuning, evaluating base model performance.
\item \textbf{Condition 1 - English Code}: Fine-tuned on English Python from The Stack Dedup.
\item \textbf{Condition 2 - Keyword-Swapped}: Per-language fine-tuning on Legesher-transpiled Python with native keywords (e.g., \texttt{if} $\rightarrow$ \texttt{如果} in Chinese).
\item \textbf{Condition 3 - Mixed Native Sources}: Per-language blend of transpiled Python, native programming languages (e.g., Wenyan for Chinese), and community code.
\item \textbf{Condition 4 - Strictly Native Code}: Combined training on code written entirely in native languages, without English-derived content.
\end{enumerate}

\subsection{Evaluation}
We evaluate on MGSM (250 examples), X-CSQA ($\sim$1K examples), and XNLI ($\sim$5K examples) in English, Chinese, Spanish, and Urdu. Each evaluation uses dual prompts: English and language-specific. Benchmarks measure math reasoning, commonsense reasoning, and natural language inference, respectively.

\section{Conclusion} 
\label{sec:conclusion}
Our experimental ladder demonstrates that native-language code provides incremental benefits over English code, with keyword-swapped and mixed native sources showing clear improvements. Strictly native code offers the strongest gains, suggesting cultural signals enhance multilingual LLM capabilities. Future work should explore cross-lingual transfer and larger-scale native code datasets.
